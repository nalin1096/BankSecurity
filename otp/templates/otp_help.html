Important Instructions*
  1. For added secuirty, you are provided with a facility of adding OTP functions. These functions will be used at the time of Login, where you will be give a random OTP generated by the bank and you have to proivde the modified OTP based on your functions for each digit.
  2. Notations used:
    1. [y] = y % 10 i.e. y modulo 10(For e.g. [81] = 1, [6] =6, [25] = 5, [-1] = 9)
    2. y^2 = y*y i.e square of y (For e.g 3^2 = 9, 7^2 = 49)
    3. y’ = [10-y] i.e. 10’s complement (For e.g. 3’ = 7, 5’ = 5, 0’ = 0, 6’= 4)
    4. y~ = [9-y] i.e. 9’s complement (For e.g. 3’ = 6, 5~ = 4, 0~ = 9, 6~=3 )
    5. d1d2 and m1m2 correspond to two digits of current day and current month, respectively. (For e.g. for the date 20 April, d1=2 d2=0 and m1=0 m2 =4)
    6. min1 and min2 represent the first minimum (smallest) and second minimum digit in the input OTP, x (for e.g. if OTP i.e. x = 192553 , min1=1 and min2 = 2. If OTP i.e. x = 600125 , min1=0 and min2=1)
  3. Some sample transformations are specified below:
    1. Sample 1: f e d c b a
    2. Sample 2: a’ b~ c’ d~ e’ f~
    3. Sample 3: [a+b] [a’*b] [c+d] [c’*d] [e+f] [e’*f]
    4. Sample 4: [a+c] [b+d] [c+e] [d+f] [e+a] [f+b]
    5. Sample 5: [[a^2]+[b^2]] [[b^2]+[c^2]] [[c^2]+[d^2]] [[d^2]+[e^2]] [[e^2]+[f^2]] [[f^2]+[a^2]]
  4. You may refer to these sample transformations and/or customize them as per your preferences, to define your own specific transformation to be used at the time of authentication, for potentially enhanced security.
*OTP functions for each digit and as a whole are optional.
